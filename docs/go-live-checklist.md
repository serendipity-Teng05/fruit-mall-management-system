# 水果商城上线工作边界与检查清单

这份清单把“代码已经具备”与“必须在真实服务器或第三方平台完成”分开，避免把本地演示成功误认为生产上线完成。

## 一、仓库内已经完成

- 前后端统一启动、分环境配置、生产配置启动校验和健康检查。
- 数据库 v2 到 v7 可重复迁移、版本记录、完整性检查、备份和显式确认恢复脚本。
- 人员、角色、权限关系规范化；关键外键、唯一约束和查询索引恢复。
- 短期访问令牌、可轮换刷新令牌、退出失效、密码加密、验证码和登录限流。
- Redis 生产 Session/限流接口、多实例定时任务数据库锁、请求 ID 和审计日志。
- 支付渠道能力由后端控制；生产未接真实适配器时拒绝支付，避免模拟支付冒充真实支付。
- 前端断网提示、统一请求错误、401 自动刷新、生产源码检查和构建门禁。
- Docker、Nginx、安全响应头、Prometheus 指标、CI 后端测试与前端构建。

## 二、正式上线前必须在外部完成（P0）

- [ ] 购买正式域名，完成 ICP 备案，配置 HTTPS 证书并强制 HTTP 跳转 HTTPS。
- [ ] 准备生产 MySQL 8 和 Redis；数据库启用 TLS，应用使用独立低权限账号，不使用 `root`。
- [ ] 通过密钥管理系统生成并保存至少 64 位随机 JWT 密钥，禁止写入仓库或聊天记录。
- [ ] 在维护窗口执行备份、v2 到 v7 迁移、`verify-database.ps1`，并确认所有问题计数为 0。
- [ ] 配置对象存储/CDN 保存商品图片，限制文件类型、大小和访问权限，避免依赖应用服务器本地磁盘。
- [ ] 选择真实支付或暂时关闭支付：真实支付必须提供商户号、应用 ID、证书/私钥、异步通知域名和退款权限。
- [ ] 接入监控与告警平台，至少覆盖 5xx、P95 延迟、JVM、数据库连接池、Redis、磁盘、支付回调和定时任务。
- [ ] 配置异机加密备份和 MySQL binlog，并在另一套环境完成一次恢复演练。
- [ ] 根据实际经营主体审核隐私政策、用户协议、退款/配送规则和联系方式；页面文案需经过人工合规确认。
- [ ] 上线前完成弱口令清理，修改所有演示账号密码，删除无用测试数据和测试管理员。

## 三、真实支付接入清单（没有资质时保持关闭）

- [ ] 微信支付 API v3 / 支付宝开放平台服务端 SDK 下单。
- [ ] 回调签名和证书校验，校验商户号、应用 ID、订单号、金额和币种。
- [ ] 以平台事件 ID 做幂等处理，不记录私钥、完整敏感报文或用户支付凭据。
- [ ] 实现主动查单、超时关单、退款、退款查询、异常补偿和每日对账。
- [ ] 在官方沙箱或最小金额真实环境完成成功、失败、重复通知、延迟通知和退款测试。

## 四、上线后建议迭代（P1/P2）

- 管理员 TOTP 双因素认证、找回密码、邮箱/手机号验证和高风险操作二次确认。
- 搜索联想、商品评价、优惠券、售后工单、物流轨迹和库存预警。
- 图片自动压缩与 WebP/AVIF、多规格 SKU、批次/保质期和采购入库。
- 管理后台操作导出、经营报表、支付对账差异处理和异常订单工作台。
- E2E 自动化覆盖注册、登录、下单、取消、收货和角色权限；定期压测与依赖升级。
- 无障碍、移动端小屏、SEO 结构化数据、真实域名 sitemap 和性能预算持续检查。

## 五、发布判定

只有第二部分 P0 全部完成，并且真实支付选择为“完整接入”或“明确关闭”时，才应标记为可公开上线。当前仓库已经具备部署基线，但域名、服务器、商户资质、生产密钥、对象存储、合规材料和监控账号不能由本地代码代替。
